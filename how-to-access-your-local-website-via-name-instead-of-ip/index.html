How to access your local website via name instead of IP

Ben July 10, 2021 network

Many people may have this experience, you write your own little web site, you try it with a local loopback address like http://127.0.0.1 or http://localhost to access your new piece of work. Many people may easily understand that 127.0.0.1 which represents your own machine, but may not understand why localhost works?

When you put the address http://localhost in your browser, how it knows it is your computer? You may say the name "localhost", which represents your own computer. But this is not the case. The request is routed via IP, not name. It is like you drive a car using GPS. The GPS guides you by an address, not a name. If you put a name, it will convert the name to an address. There is a mapping between the address and the name. This applies to our case as well. When the http request is sent, it will convert the name "localhost" to an address and use the default port 80 to send the request to your http server.

On Windows platform, this local mapping is done via hosts file: (Other System like Linux and MacOS has hosts file as well in a different location)
C:\Windows\System32\drivers\etc\hosts

The mapping is like this:

127.0.0.1       localhost

Sometimes, the mapping between "127.0.0.1" and "localhost' are commented out, but this should be a built-in mapping. You can add a mapping yourself, like this:

127.0.0.1    hello.world

After the mapping is saved, you can access your local web server via: http://hello.world

Please note that you should have a running web server with default port 80 to make it work. If you do not have one and want to test it, you can download a web server software like apchace or if you know Python, you can do this trick simply like this:

python -m http.server 80 --bind 127.0.0.1

You can always change the mapping to whatever you like. Whenever a request is sent, it will look up this hosts file first, if a mapping is found, it will use the found IP, otherwise it will ask your DNS server to find it for you. Please note, do not change this name to a known Internet website, especially one without https, otherwise you will not be able to access this website anymore.
